## Том - 0

### 📌 Описание проекта
Tibicen — cистема управления интернет-магазином. Поддерживает операции управления товарами, заказами, пользователями и отображает витрину для покупателей. Реализован как монолит на NestJS с разделением на REST API и BFF endpoint'ы для фронтенда. Angular используется для рендера страниц на витрине.

**Основные характеристики:**
- Архитектура: Monolith + BFF слой
- Backend: NestJS (Modular Architecture)
- Frontend: Angular
- База данных: PostgreSQL

### 👥 Описание основных пользователей
1. **Customer** – конечный покупатель
   - Просматривает товары, добавляет в корзину, оформляет заказы
2. **Manager (admin)** – оператор магазина
   - Управляет товарами и заказами
3. **SuperAdmin** *(в будущем)* – полный контроль, включая управление staff-пользователями

### 🗂️ Структура страниц магазина
1. **Admin Panel** — страницы для управления магазином
2. **Storefront** — страницы для формирования заказов пользователем

## Том - 1

### 🛠️ Admin Panel

#### 📦 Страница управления продуктами

- **Описание:** Отображает список всех товаров с возможностью фильтрации, создания, редактирования и удаления. Поддерживает открытие карточки товара.

- **Пользователи:**
  - `Manager`:
    - Просмотр списка
    - Создание товара
    - Изменение товара
    - Удаление товара

- **FE route:** `/admin/products`
- **BFF endpoint:** `GET /admin/products` → [`ProductManagementBFF`](#productmanagementbff)

- **Используемые модули:**
  - [`Product`](#product-module)
  - [`Category`](#category-module)
  - [`Auth`](#authmodule)

## Том - 2

### 🧠 AdminProducts

- **Назначение:** Обеспечивает передачу и агрегацию данных для `/admin/products`
- **Формат ответа:**
```ts
{
  items: [
    {
      sku: string;
      id: UUID;
      name: string;
      img_preview: string;
      images: string[];
      quantity_on_stock: number;
      description: string;
      category: string;
    }
  ];
  total_items_count: number;
}
```
- **Зависимости:**
  - [`Product`](#product-module)
  - [`Category`](#category-module)
  - [`Auth`](#authmodule)
  - [`Users`](#user-module)

---

### 👤 Users Module

- **Назначение:** Управление пользователями (Customer и Staff), регистрация, авторизация, CRUD

- **REST Эндпоинты:**
  - `GET /api/users/:id` — найти по ID
  - `GET /api/users` — список пользователей (`?limit=&offset=`)
  - `POST /api/users` — создать пользователя
  - `PATCH /api/users/:id` — обновить пользователя
  - `DELETE /api/users/:id` — удалить пользователя

---

### 🔐 AuthModule
Модуль аутентификации. Поддерживает JWT, Guard’ы, роли пользователей.

### 📦 Product Module
CRUD логика для товаров, поиск по фильтрам, работа с изображениями.

### 🗂️ Category Module
Работа с категориями товаров. CRUD, фильтрация, отображение и вложенность.

### 📦 Stock Module
Модуль склада: остатки, движения, резервы, доступность.
